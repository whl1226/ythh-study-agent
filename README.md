# 「研途护航」考研考公 AI 智能督学 Agent — 项目说明文档

## 一、项目概述

### 1.1 项目背景

每年数百万学生参加考研和考公考试，但备考过程中普遍面临以下痛点：

- **缺乏个性化规划**：不同学生基础不同、目标不同，但备考方案千篇一律
- **自律性不足**：缺少有效的督学机制，容易拖延和放弃
- **信息不对称**：院校/岗位数据分散，择校择岗决策盲目
- **答疑资源有限**：无法随时随地获得针对性的学业指导

### 1.2 解决方案

**研途护航** 是一套基于 **AI Agent 架构**的智能督学系统，通过 6 个专职 AI 智能体协同工作，为备考学生提供从 **测评 → 规划 → 督学 → 学习 → 答疑** 的全流程个性化支持。

### 1.3 核心理念

```
测（知己录）→ 规（运筹策）→ 督（砥砺阁）→ 学（博观苑）→ 问（解惑堂）
```

每个环节由独立的 AI Agent 负责，既各司其职，又数据互通，形成完整的备考闭环。

## 二、功能模块详解

### 2.1 备考档案 Agent（知己录）— Profile

**定位**：备考画像与能力评估

通过 6 步引导式问卷，快速建立学生的个性化备考档案：

| 步骤 | 采集内容 | 说明 |
|:---:|---------|------|
| 1 | 考试类型 | 考研 / 考公 |
| 2 | 目标院校/岗位 | 支持从知识库选择，提供竞争比参考 |
| 3 | 各科自评成绩 | 考研 4 科 / 考公 6 模块 |
| 4 | 优势与薄弱项 | 定性分析 |
| 5 | 每日可用学习时间 | 计算总可用学时 |
| 6 | 作息安排与考试日期 | 倒计时与时间规划基础 |

**AI 输出**：
- 雷达图数据（可视化能力分布）
- 个性化标签（如"英语薄弱型""长线备考型"）
- 综合分析摘要（100 字以内）
- 针对性备考建议（每科 50 字以内）

**技术实现**：
- 前端：6 步向导式表单，支持考研/考公双模式切换
- 后端：调用 AI 生成结构化 JSON（雷达数据 + 标签 + 文本）
- 图表：Chart.js 雷达图，支持目标分数线叠加显示

---

### 2.2 备考规划 Agent（运筹策）— Planner

**定位**：四阶段备考计划生成与管理

AI 自动生成符合备考规律的四阶段计划：

```
基础阶段（夯实基础）
  ↓
强化阶段（重点突破）
  ↓
冲刺阶段（查漏补缺）
  ↓
模拟阶段（实战演练）
```

**每个任务包含**：
- 知识点级标题（如"高等数学·极限与连续"）
- 详细描述（50-100 字）
- 参考资源（2-3 个）
- 量化验收标准（如"完成 50 道真题，正确率 ≥ 80%"）
- 备考贴士

**附加功能**：
- 科目时间分配饼图
- 备考旅程时间线可视化
- 逐阶段生成（支持用户自定义需求）
- 任务打卡与状态管理
- AI 动态计划调整（根据完成情况重新规划）

**技术实现**：
- Prompt 工程：两个专用 Prompt 模板（全阶段 / 单阶段），支持 `{{variable}}` 变量注入
- JSON 结构化输出：AI 返回标准 JSON，前端解析渲染
- 任务状态：完成/未完成切换，实时同步后端

### 2.3 督学面板 Agent（砥砺阁）— Supervisor

**定位**：日常督学主面板，系统首页

这是用户日常使用最频繁的模块，集成多项督学功能：

#### 2.3.1 每日激励
- 55 条励志名言自动轮播（`/api/supervisor/quotes`）
- 个性化问候语（根据时间段：早上好/下午好/晚上好）
- 距离考试倒计时

#### 2.3.2 四项核心指标
| 指标 | 说明 |
|------|------|
| 考试倒计时 | 距离目标考试的天数 |
| 连续打卡 | 连续完成任务的天数 |
| 本周学时 | 本周累计学习时长 |
| 目标完成度 | 已完成任务 / 总任务数 |

#### 2.3.3 今日任务
- 展示当日待完成任务列表
- 勾选完成，实时更新统计

#### 2.3.4 番茄钟
- 可配置专注时长（默认 25 分钟）和休息时长（默认 5 分钟）
- SVG 环形进度条
- 可关联具体任务
- 自动记录番茄钟会话

#### 2.3.5 学习光谱
- 自定义 SVG 绘制的 30 天活跃度环形图
- 颜色深浅表示学习强度
- 直观展示学习连续性

#### 2.3.6 趋势分析
- 7 天 / 30 天切换的学习趋势折线图
- 多数据集：学习时长、任务完成数

#### 2.3.7 成就系统
- 8 枚成就徽章（首次打卡、连续 7 天、完成 10 个任务等）
- 达成时触发撒花庆祝动画

#### 2.3.8 AI 周报
- 一键生成本周学习总结
- AI 输出：总体评分（0-100）、亮点、不足、建议、下周重点、鼓励语

#### 2.3.9 数据管理
- 学习数据导出（JSON 格式备份）
- 数据导入恢复

**技术实现**：
- Canvas 粒子系统实现撒花动画
- SVG 自定义绘制学习光谱和番茄钟进度环
- Chart.js 折线图支持数据集切换
- 计数动画（countUp）实现数字跳动效果

### 2.4 备考智库 Agent（博观苑）— Knowledge

**定位**：基于 RAG 的智能知识问答系统

这是系统中技术含量最高的模块，采用 **RAG（Retrieval-Augmented Generation，检索增强生成）** 架构。

#### 五大功能标签页

| 标签页 | 功能 | 数据来源 |
|--------|------|---------|
| 择校推荐 | 985/211/双一流/双非院校数据，冲/稳/保三档推荐 | `exam_graduate.json` |
| 岗位推荐 | 国考/省考岗位筛选，含竞争比、进面分、薪资 | `exam_civil.json` |
| 风险分析 | 压分院校预警、歧视性政策、黑名单岗位 | `exam_risks.json` |
| 考纲查询 | 各科目考纲、分值占比、高频考点、推荐资料 | `exam_syllabus.json` |
| 自由问答 | 开放式 AI 问答，自动注入知识库上下文 | 全部知识库 |

#### RAG 检索流程

```
用户输入 → 中文分词 → 意图识别（9 种模式）→ 本地知识库匹配 → 评分排序
                                                              ↓
用户输入 + 检索结果 → 构造 RAG Prompt → AI 生成个性化回答
```

**意图识别覆盖**：考研咨询、考公咨询、院校查询、岗位查询、风险查询、大纲查询、学习方法、推荐请求、通用问答

**技术实现**：
- 中文分词器：按中文字符和英文/数字分别切分
- 意图检测：9 种正则表达式模式匹配
- 评分排序：关键词长度加权 + 字段匹配分数
- 流式输出：SSE 实时推送 thinking → chunk → done
- 知识库缓存：1 分钟 TTL，避免重复加载

---

### 2.5 智能答疑 Agent（解惑堂）— Tutor

**定位**：多模式 AI 对话答疑

四种对话模式，适配不同学习场景：

| 模式 | 教学理念 | 交互方式 |
|------|---------|---------|
| 苏格拉底式 | 以问代讲，引导自主思考 | AI 提问 → 学生回答 → AI 追问 |
| 限时测验 | 练习巩固，检验掌握程度 | AI 出题 → 学生作答 → AI 解析 |
| 计划制定 | 行动导向，分解学习目标 | 描述需求 → AI 生成短期计划 |
| 综合指导 | 策略 + 情绪双维度支持 | 提问 → AI 分析 + 建议 + 鼓励 |

**智能特性**：
- **情绪检测**：根据用户输入判断情绪状态（中性/鼓励/安抚/庆祝），调整回复语气
- **思维导图**：对知识结构类问题，AI 生成 Markdown 格式的思维导图
- **追问建议**：每次回复后生成 3 个追问按钮，引导深入学习
- **对话持久化**：保留最近 100 条对话历史

**技术实现**：
- SSE 流式对话：XHR 进度事件逐块解析
- 思考过程展示：`thinking` 类型事件实时显示 AI 推理过程
- Markdown + KaTeX 渲染：支持数学公式（如 `$\int_0^1 f(x)dx$`）
- 预设问题轮播：每种模式 10 个预设问题，降低使用门槛

---

### 2.6 课程表 Agent — Schedule

**定位**：AI 智能排课与课程管理

**时间节次**（参照宁夏师范学院格式）：

| 节次 | 时间 |
|------|------|
| 第 1-2 节 | 08:00 - 09:40 |
| 第 3-4 节 | 10:00 - 11:40 |
| 第 5-6 节 | 14:30 - 16:10 |
| 第 7-8 节 | 16:30 - 18:10 |
| 第 9-10 节 | 19:30 - 21:10 |

**科目体系**：
- 考研：政治、英语、数学、专业课
- 考公：常识判断、言语理解、数量关系、判断推理、资料分析、申论

**功能特性**：
- AI 一键生成整周课程表
- 手动编辑模式：单格编辑（选择科目 + 填写活动描述）
- 颜色编码：按科目自动着色
- 当前节次高亮
- 今日课程集成到督学面板
- 30 秒生成倒计时动画

**容错机制**：AI 生成失败时，自动降级为基于规则的兜底排课方案

### 3.2 技术选型与理由

| 层级 | 选型 | 选择理由 |
|------|------|---------|
| **前端框架** | 原生 JavaScript (IIFE) | 零依赖，加载快，适合课程项目演示 |
| **样式方案** | CSS3 + CSS Variables | 主题统一管理，无需预处理器 |
| **图表库** | Chart.js 4.4 | 轻量、易用、图表类型丰富 |
| **数学渲染** | KaTeX 0.16.11 | LaTeX 公式渲染，支持考研数学场景 |
| **Markdown** | Marked.js 12.0 | AI 回复的富文本渲染 |
| **后端框架** | Express 4.21 | Node.js 生态成熟，轻量灵活 |
| **数据存储** | SQLite (sql.js) | WASM 运行，零原生依赖，7 张表，自动迁移旧 JSON 数据 |
| **AI 接入** | OpenAI 兼容协议 | 国产模型（豆包/DeepSeek）均支持 |
| **流式传输** | SSE (Server-Sent Events) | 比 WebSocket 更轻量，单向推送足够 |

### 3.3 前端架构设计

**模块化方案**：IIFE + window 命名空间

```javascript
// 每个模块都是一个 IIFE，挂载到 window
window.Profile = (function() {
  // 私有变量和函数
  function render() { ... }
  function handleNext() { ... }

  // 公开 API
  return { render, init };
})();
```

**状态管理**：发布-订阅模式

```javascript
// Store 核心 API
Store.set('user', userData);          // 设置状态
Store.get('user');                     // 读取状态
Store.subscribe('user', callback);    // 订阅变更
Store.persist();                       // 持久化到 localStorage
```

**路由守卫**：
- 未登录 → 重定向到登录页
- 未完成测评 → 重定向到备考档案页

### 3.4 后端架构设计

**分层架构**：

```
路由层 (routes/)     → 处理 HTTP 请求，参数校验
  ↓
服务层 (services/)   → 业务逻辑，AI 调用，数据存取
  ↓
存储层 (db.js)       → SQLite 数据库（sql.js WASM），7 张表
```

**数据库表结构**（`server/services/db.js`）：

| 表名 | 用途 | 关键字段 |
|------|------|---------|
| `users` | 用户账号 | uid, username, password (SHA-256) |
| `session` | 会话状态（单行约束） | current_uid, login_at, expires_at |
| `config` | 键值配置 | key, value (JSON) |
| `user_data` | 用户数据（画像/计划/对话/课程表） | uid, profile, plan, conversations, schedule |
| `checkins` | 每日打卡记录 | uid, date, completed_tasks, study_hours, mood |
| `pomodoro_sessions` | 番茄钟会话 | uid, date, timestamp, duration, type |
| `supervisor_reviews` | AI 周报/月报 | uid, review_type, data (JSON), generated_at |

首次启动时自动执行 JSON → SQLite 一次性迁移。

**AI 服务双模式**：

```
请求进入
  │
  ├─ 有 API Key → 调用真实 AI 接口
  │                ├─ 豆包：/api/v3/bots/chat/completions
  │                └─ DeepSeek：/v1/chat/completions
  │
  └─ 无 API Key → Mock 模式
                   └─ 返回硬编码的模拟响应
```

### 3.5 知识检索引擎

```
查询输入
  │
  ▼
中文分词 (逐字符切分中文 + 连续切分英文数字)
  │
  ▼
意图识别 (9 种正则模式)
  │  ├─ 考研意图 → 搜索院校库
  │  ├─ 考公意图 → 搜索岗位库
  │  ├─ 风险意图 → 搜索风险库
  │  ├─ 大纲意图 → 搜索考纲库
  │  └─ ...
  ▼
关键词匹配 + 评分排序
  │  ├─ 字段权重：名称 > 标签 > 描述
  │  └─ 关键词长度加权
  ▼
返回 Top-N 结果 → 注入 AI Prompt
```

## 四、项目亮点

### 4.1 AI Agent 架构设计

系统采用 **6 个专职 Agent 协同**的架构，而非单一的大模型对话。每个 Agent：
- 有独立的 Prompt 模板（`server/prompts/*.txt`）
- 有独立的前端模块（`js/agents/*.js`）
- 有独立的后端路由（`server/routes/*.js`）
- 职责边界清晰，可独立迭代

### 4.2 RAG 检索增强生成

知识问答模块不是简单的"把问题丢给 AI"，而是：
1. 先从本地知识库检索相关数据
2. 将检索结果注入 Prompt 作为上下文
3. AI 基于真实数据生成个性化回答

这保证了回答的**准确性**（有数据支撑）和**个性化**（结合学生档案）。

### 4.3 流式对话体验

通过 SSE（Server-Sent Events）实现打字机效果：
- `thinking` 事件：展示 AI 推理过程
- `chunk` 事件：逐块输出回答内容
- `done` 事件：标记回答完成

相比等待完整响应再展示，流式输出大幅提升了用户体验。

### 4.4 零配置即用

- 无需额外安装数据库（SQLite 通过 sql.js WASM 运行，数据文件自动生成）
- 无需配置构建工具
- 无 API Key 时自动降级为 Mock 模式
- 首次启动自动从旧版 JSON 文件迁移数据
- Windows 用户双击 `启动.bat` 即可运行

### 4.5 丰富的可视化

- **雷达图**：能力维度分布（Chart.js）
- **饼图**：科目时间分配（Chart.js）
- **折线图**：学习趋势（Chart.js）
- **热力图**：30 天活跃度
- **SVG 光谱环**：自定义绘制的学习活跃度
- **SVG 番茄钟**：环形进度条
- **撒花动画**：Canvas 粒子系统
- **LaTeX 公式**：KaTeX 渲染

## 五、运行指南

### 5.1 环境要求

- **Node.js** ≥ 16（推荐 18 LTS）
- **操作系统**：Windows / macOS / Linux

### 5.2 启动方式

**方式一：双击启动（Windows）**

```
双击「启动.bat」
```

自动检测环境 → 安装依赖 → 启动服务器 → 3 秒后打开浏览器

**方式二：命令行**

```bash
npm install
npm start
# 访问 http://localhost:3000
```

### 5.3 加载演示数据

```bash
node seed-demo-data.js
```

为用户 `111`（密码 `111`）生成 60 天模拟数据（含学习记录、AI 对话、番茄钟会话、周报等）。

```jsonc
  },
  "session": {
    "duration": 86400000            // 会话有效期（毫秒），默认 24 小时
  }
}
```

不配置时自动使用 Mock 模式，所有功能均可演示。

## 六、已知局限与改进方向

| 当前局限 | 改进方向 |
|---------|---------|
| 单用户会话（session 单行约束） | 改用 JWT Token + 多用户支持 |
| sql.js WASM 性能有限 | 高并发场景迁移至 better-sqlite3 或 PostgreSQL |
| 关键词匹配检索 | 引入向量数据库（如 FAISS / Milvus）实现语义检索 |
| 原生 JS 无框架 | 可迁移至 Vue / React 提升可维护性 |
| 无自动化测试 | 添加 Jest 单元测试 + Playwright E2E 测试 |
| 无容器化部署 | 添加 Dockerfile + docker-compose |

## 八、总结

**研途护航** 是一套功能完整、架构清晰的 AI 智能督学系统原型。它展示了：

1. **AI Agent 多智能体协作**的实际应用（6 个专职 Agent 协同工作）
2. **RAG 检索增强生成**在教育场景的落地（本地知识库 + 意图识别 + 评分排序）
3. **SSE 流式传输**带来的实时交互体验（含思考过程展示）
4. **Prompt 工程**在多模式对话中的设计技巧（6 套专用模板）
5. **前端工程化**在无框架约束下的模块化实践（IIFE + 发布-订阅状态管理）
6. **SQLite 数据库**的实际应用（sql.js WASM 运行，零原生依赖，自动迁移）

项目从需求分析、架构设计、AI 对接、前端开发到知识库构建，完整覆盖了一个 AI 应用的全生命周期，可作为 AI Agent 在教育领域应用的参考实现。
